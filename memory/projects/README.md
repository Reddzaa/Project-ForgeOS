# Project history

One file per completed (shipped, parked, or killed) project — written by the
memory-consolidator skill after the retro. This folder is the permanent
record the idea-filter and profile patterns draw on. Killed projects belong
here just as much as shipped ones.

## File format

```markdown
# {project-name}

**Type:** {desktop-utility | discord-bot | ...} · **Stack:** {slug}
**Outcome:** {shipped | parked | killed} ({YYYY-MM}) · **Duration:** {rough}
**One-liner:** {what it was}

## How it went

{3–6 lines: what shipped, what didn't, why it ended the way it did.}

## Key decisions and their facit

- D-001 {title} — {held | failed}: {one line}

## Lessons produced

- [[lessons/{lesson-file}]]
```

## Rules

- Entries are never deleted — a project history with gaps can't show
  patterns.
- Keep the facit honest: "held so far" and "failed" are both useful; blank is
  not.
