# Profile

A few files about the developer, not the projects: stack defaults, working
style, and recurring patterns. Skills read this to tailor recommendations
(stack-picker reads preferences; idea-filter and plan-project read patterns).

## Files

- `preferences.md` — stack defaults, environment, agents in use. Edited by
  hand or updated (with confirmation) by the memory-consolidator.
- Pattern insights live in `preferences.md` under "Patterns" until they're
  numerous enough to deserve their own file.

## Rules

- **Patterns are guidance, never judgment.** Write "prove hosting in Phase 0
  for bots — historically the stall point", not "you always give up at
  hosting". A profile that scolds gets ignored.
- Every pattern cites the project history that shows it
  (`[[projects/...]]`).
- The human confirms every pattern before it's written — the
  memory-consolidator suggests, the owner decides what the system believes
  about them.
